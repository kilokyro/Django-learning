Django 練習測驗

初始化

建立名稱為 django-todo-list 的專案，並使用 poetry 進行虛擬環境與套件的管理

需安裝的套件（都用最新版即可）

  • base

    • django
    • django-extensions

  • dev

    • djlint
    • black
    • isort
    • ipython

Model

建立 APP 名稱為 todo

建立以下資料表

  • Project

    • name (char field: max_length=255, unique=true)

  • Tag

    • name (char field: max_length=255, unique=true)

  • Task

    • title (char field: max_length=255)
    • content (text field)
    • is_finish (bool field: default=false)
    • project (FK => Project, on_delete=protect)
    • tags (many to many => Tag)

P.S. 提示：別忘了建好 Model 後要套用到 DB 中

Admin

將以上兩張表都註冊進 Admin 系統中

  • Project

    • 列表中顯示欄位為 name

  • Tag

    • 列表中顯示欄位為 name

  • Task

    • 列表中顯示欄位為 content 與 is_finish
    • 需可透過 title, content 進行搜尋
    • 需可透過 is_finish, project, tags 進行篩選

Template

畫面規劃如下

  • base.html 提供專案中所有 APP 的基礎畫面（需顯示 message）
  • todo_base.html 提供繼承 base.html 並顯示 H1 標題 TODO List

P.S. 其餘畫面檔案名稱與 views 相同

URL

路由設定如下

路徑 名稱 說明
/todo/tasks/ task_list 任務列表
/todo/tasks/create/ task_create 建立任務
/todo/tasks/<pk>/ task_retrieve 單一任務
/todo/tasks/<pk>/update/ task_update 更新任務
/todo/tasks/<pk>/delete/ task_delete 任務刪除

View

名稱與用途如下

  • task_list: 列出所有任務（需有連結到 task_create 與 task_retrieve
  • task_create: 讓使用者建立任務（完成後回到 task_list）
  • task_retrieve: 讓使用者根據 PK 取得任務（需有連結到 task_list, task_update 與 task_delete）
  • task_update: 讓使用者根據 PK 更新任務（完成後回到 task_retrieve）
  • task_delete: 讓使用者根據 PK 刪除任務（需透過 Form 確認，完成後回到 task_list）

P.S. 以上畫面隨意設計，完成動作後需顯示 message

加分項

將 Project 與 Tag Model 也都建立 CRUD
